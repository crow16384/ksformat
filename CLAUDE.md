## Package Purpose

`ksformat` provides SAS PROC FORMAT-like functionality for R:
- Value-to-label mappings (`ks_format`) and label-to-value mappings (`ks_invalue`)
- Proper handling of NA, NULL, NaN and other missing values
- Range-based formatting for numeric data
- SAS-style datetime formatting
- Global format library for reusable format definitions

## Package Structure

Standard CRAN-style R package:

- `R/` — All R function definitions
  - `format_create.R` — `fnew()`, format creation and printing
  - `format_apply.R` — `fput()`, `fputn()`, `fputc()`, `fput_all()`, `fput_df()`
  - `format_invalue.R` — `finput()`, `fnew_bid()`, invalue logic
  - `format_datetime.R` — `fnew_date()`, SAS datetime format support
  - `format_parse.R` — `fparse()`, `fexport()`, `fimport()` (CNTLOUT CSV)
  - `utilities.R` — `is_missing()`, `range_spec()`, library management
- `man/` — `.Rd` documentation (auto-generated by roxygen2)
- `tests/testthat/` — test suite (~130+ test cases)
- `DESCRIPTION` — package metadata (version 0.3.1, MIT license)
- `NAMESPACE` — exports (auto-generated by roxygen2)

## Core API

| Function | Purpose |
|---|---|
| `fnew(...)` | Create format (value → label) |
| `fput(x, fmt)` | Apply format to vector |
| `fputn(x, fmt)` | Apply format, numeric input |
| `fputc(x, fmt)` | Apply format, character input |
| `fput_all(x, fmt)` | Return all matching labels |
| `fput_df(data, ...)` | Apply formats to data frame columns |
| `finput(x, inv)` | Apply invalue (label → value) |
| `finputn(x, inv)` | Invalue with numeric output |
| `finputc(x, inv)` | Invalue with character output |
| `fnew_bid(...)` | Create bidirectional format + invalue |
| `fnew_date(pattern)` | Create SAS-style datetime format |
| `fparse(text)` | Parse format from text definition |
| `fexport(fmt)` | Export format to CNTLOUT CSV |
| `fimport(file)` | Import format from CNTLOUT CSV |
| `fprint(fmt)` | Print format definition |
| `fclear()` | Clear format library |
| `is_missing(x)` | Check for NA/NULL/NaN |
| `range_spec(low, high, label)` | Create range specification |

## Missing Value Priority

1. NA / NULL / NaN → `.missing` label (if defined), else NA
2. Exact match → mapped label
3. Range match → range label
4. No match → `.other` label or original value

Special parameters: `keep_na`, `na_if`, `include_empty` (see `is_missing()`).
